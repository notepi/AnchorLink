# 商业航天情报系统 - 业务方案评审

> 版本：v1.0 | 最后更新：2026-04-01
> 本文档面向业务专家，用于评估系统方案设计的合理性。

---

## 1. 业务背景

### 1.1 投资标的概述

**铂力特（688333.SH）**：
- 国内金属增材制造（3D打印）龙头
- 科创板上市，商业航天核心标的
- 主营：航空航天零部件制造服务

**行业特征**：
- 商业航天板块情绪联动强
- 受政策催化影响大
- 主力资金流向是重要信号

### 1.2 业务痛点

| 痛点 | 具体问题 | 影响 |
|------|----------|------|
| 信息分散 | 行情、资金、新闻数据分散在多个平台 | 复盘耗时 |
| 缺乏参照 | 不知道铂力特在商业航天板块中的相对位置 | 难以判断强弱 |
| 主力难跟踪 | 资金流向数据专业性强，难以快速理解 | 错过资金信号 |
| 连续性缺失 | 只有当日数据，无法判断趋势连续性 | 误判趋势 |

### 1.3 解决目标

**核心目标**：将"每天花30分钟看盘+复盘"压缩为"5分钟看一份报告"。

**具体目标**：
1. 自动化数据获取（替代手动查数据）
2. 结构化分析（替代主观判断）
3. 生成可读报告（替代零散记录）

## 2. 方案设计

### 2.1 整体架构

```
数据获取 → 指标计算 → 连续观察 → 综合诊断 → 报告输出
```

**三条数据线**：

| 数据线 | 功能 | 输出 |
|--------|------|------|
| 行情数据线 | 获取行情、计算指标 | 85个指标 |
| 新闻数据线 | 抓取新闻、提取事件 | 事件列表 |
| 日报模块 | 格式化输出 | Markdown报告 |

### 2.2 数据来源设计

| 数据类型 | 来源 | 获取频率 |
|----------|------|----------|
| 日线行情 | Tushare | 每日收盘后 |
| 资金流向 | Tushare (moneyflow) | 每日收盘后 |
| 基本面 | Tushare (daily_basic) | 每日收盘后 |
| 新闻事件 | 配置化源（待实现） | 按需 |

**选择理由**：
- Tushare：数据全面、接口稳定、免费额度充足
- 配置化新闻源：灵活扩展，不硬编码

### 2.3 指标体系设计

**四层架构**：

```
L1: 基础数据（Tushare 原始字段）
    ↓ 涨跌幅、成交额、主力净流入、PE/PB...
L2: 衍生指标（计算得出）
    ↓ 相对强弱、成交额/5日均值、板块排名...
L3: 状态标签（规则化映射）
    ↓ 价格强度、成交额强度、综合信号...
L4: 综合诊断（多维度综合）
    → 诊断标签、原因、观察清单
```

### 2.4 报告输出设计

**报告结构**：

| 章节 | 内容 | 用途 |
|------|------|------|
| 状态面板 | 8个状态标签 | 快速定位 |
| 今日结论 | 诊断标签+原因+下一步 | 决策依据 |
| 证据板 | 关键指标数值 | 验证结论 |
| 资金流向 | 主力行为分析 | 资金判断 |
| 近5日结构 | 连续性分析 | 趋势判断 |
| 估值与位置 | PE/PB、板块排名 | 估值参考 |
| 关键信号 | 异常/积极/风险 | 信号识别 |
| 明日观察 | 观察清单 | 下一步行动 |

## 3. 设计理由

### 3.1 为什么选择"相对强弱"而非绝对涨跌

**问题**：铂力特涨了2%，这算强还是弱？

**分析**：
- 如果板块涨了5%，铂力特涨2% → 跑输板块 → 弱
- 如果板块跌了3%，铂力特涨2% → 跑赢板块 → 强

**结论**：绝对涨跌受大盘和板块影响，无法反映个股真实强弱。

**设计**：
```
relative_strength = anchor_return - sector_avg_return
```

**价值**：剔除了大盘和板块因素，聚焦个股因素。

### 3.2 为什么设计"双层股票池"

**问题**：哪些股票应该作为铂力特的对标？

**分析**：
- 有些股票与铂力特情绪联动强（如航天电子）
- 有些股票与铂力特产业链相关（如华曙高科）
- 有些股票相关性弱（如军工电子）

**设计**：

| 层级 | 用途 | 参与均值计算 | 典型股票 |
|------|------|-------------|----------|
| core_universe | 交易层对标 | 是 | 航天动力、航天电子、中国卫星 |
| research_core | 研究层对标 | 否 | 华曙高科、中天火箭 |
| extended_universe | 扩展观察 | 否 | 军工电子类 |

**价值**：分层对标，避免噪音干扰。

### 3.3 为什么设计"连续观察层"

**问题**：单日跑赢板块，能判断趋势吗？

**分析**：
- 单日跑赢可能是噪音
- 连续3日跑赢更可信
- 连续跑赢+连续放量=趋势确认

**设计**：
- 近5日跑赢天数
- 连续跑赢/跑输天数
- 连续放量/缩量天数
- 资金连续流入/流出天数

**价值**：判断趋势连续性，避免单日噪音。

## 4. 指标设计详解

### 4.1 价格强度标签

**目的**：判断当日价格表现的强弱。

**输入**：
- anchor_return：当日涨跌幅
- relative_strength：相对强弱
- return_rank_in_sector：板块排名

**规则**：

| 条件 | 标签 |
|------|------|
| 涨幅 > 2% | 强 |
| 相对强弱 > 1% 且 排名前半 | 强 |
| 跌幅 > 2% | 弱 |
| 相对强弱 < -1% 且 排名后半 | 弱 |
| 其余 | 中 |

**设计理由**：
- 阈值固化（2%、1%），避免主观判断
- 排名作为辅助验证，避免单一指标误判

### 4.2 成交额强度标签

**目的**：判断当日成交额的活跃度。

**输入**：
- amount_vs_5d_avg：成交额/5日均值
- amount_20d_high：是否创20日新高
- amount_rank_in_sector：板块排名

**规则**：

| 条件 | 标签 |
|------|------|
| 创20日新高 | 强 |
| 成交额 > 1.5倍均值 | 强 |
| 排名前半 | 强 |
| 成交额 < 0.7倍均值 且 未新高 且 排名后半 | 弱 |
| 其余 | 中 |

**设计理由**：
- 创新高是强势信号，优先判断
- 均值比较反映相对活跃度
- 排名作为辅助验证

### 4.3 综合信号标签

**目的**：综合价格和成交额判断整体信号。

**输入**：价格强度 × 成交额强度矩阵

**规则**：

| 价格\成交额 | 强 | 中 | 弱 |
|------------|-----|-----|-----|
| 强 | 强 | 中性偏强 | 中性偏强 |
| 中 | 中性偏强 | 中性 | 中性偏弱 |
| 弱 | 中性偏弱 | 中性偏弱 | 弱 |

**设计理由**：
- 价强量强 = 确认强势
- 价强量弱 = 强势存疑
- 价弱量强 = 可能见底或资金出逃
- 价弱量弱 = 确认弱势

### 4.4 近5日结构

**目的**：判断趋势连续性。

**输入**：
- 最近5日的相对强弱序列
- 最近5日的成交额序列
- 最近5日的资金流向

**输出**：

| 指标 | 含义 |
|------|------|
| 跑赢板块 X/5 日 | 胜率 |
| 平均相对强弱 | 幅度 |
| 量能状态 | 连续放量/连续缩量/平稳 |
| 资金状态 | 偏强/有改善/偏弱 |

**判断规则**：
- 胜率 ≥ 4/5：胜率较高
- 胜率 3/5：胜率尚可
- 胜率 ≤ 2/5：胜率偏低
- 连续缩量 ≥ 2日：量能萎缩
- 连续放量 ≥ 2日：量能放大

### 4.5 诊断层

**目的**：综合所有指标输出可执行的诊断结论。

**输入**：当日指标 + 近5日指标

**输出**：

| 输出 | 示例 |
|------|------|
| 诊断标签 | "中性偏弱，下行压力仍存" |
| 诊断原因 | ["跑输板块", "近5日结构未坏", "主力未确认回流"] |
| 明日观察清单 | ["主力是否转正", "研究层对标是否继续走弱"] |
| 信号拆解 | 异常/积极/风险 |

**设计理由**：
- 标签+原因：结论+依据
- 观察清单：下一步行动
- 信号拆解：多维度信号识别

## 5. 合理性分析

### 5.1 指标设计合理性

| 维度 | 分析 |
|------|------|
| 完整性 | 覆盖价格、量能、资金、估值四大维度 |
| 可解释性 | 每个指标有明确的业务含义和规则 |
| 可验证性 | 规则固化，结果可复现 |

**潜在问题**：
- 阈值（如2%、1%）是经验值，未经验证
- 未考虑个股特性（如波动率差异）

### 5.2 数据源选择合理性

| 维度 | 分析 |
|------|------|
| 稳定性 | Tushare 是成熟的数据服务，接口稳定 |
| 成本 | 免费额度充足，适合个人投资者 |
| 覆盖 | 行情、资金、基本面全覆盖 |

**潜在问题**：
- 单一数据源，无交叉验证
- 新闻数据待实现

### 5.3 输出形式合理性

| 维度 | 分析 |
|------|------|
| 可读性 | Markdown格式，易读易分享 |
| 结构化 | 固定章节，便于快速定位 |
| 可执行 | 观察清单提供下一步行动 |

**潜在问题**：
- 非实时，仅每日更新
- 无历史回溯功能

## 6. 风险评估

### 6.1 数据风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 数据源中断 | 无法获取数据 | 建立本地缓存 |
| 数据错误 | 指标计算错误 | 数据校验层 |
| 数据延迟 | 报告延迟生成 | 监控提醒 |

### 6.2 模型风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 阈值不合理 | 误判强弱 | 定期回测优化 |
| 规则固化 | 无法适应市场变化 | 预留调整机制 |
| 因子缺失 | 遗漏重要信号 | 持续迭代 |

### 6.3 业务风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 过度依赖 | 用户盲目跟随信号 | 强调"辅助决策"定位 |
| 延迟决策 | 等待报告错过时机 | 明确定位是复盘而非实时 |
| 误用结论 | 将"中性"理解为"持有" | 提供使用指南 |
