# AnchorLink 产品设计文档

> 锚定联动分析系统 — 锚定一家公司，与板块联动对比分析。
> 当前标的：铂力特 (688333.SH)

---

## 系统总览

AnchorLink 解决的核心问题是：**"这只股票今天的表现，到底是它自己的事，还是板块的事？"**

传统看盘方式把个股和板块割裂看，但 A 股的板块效应极强 — 个股 70% 的波动可以用板块解释。如果你不知道今天是"板块带起来的"还是"个股独立走出来的"，所有买卖决策都是在瞎赌。

AnchorLink 的做法是：锚定一只股票，同时观察它所属的多个池子（产业链、主题、交易联动），从池子状态推导出个股的"处境"，再用历史相似处境做路径参考。

### 数据流向

```
原始行情 (Tushare)
  → 池状态计算 (4个池分别算)
  → 个股定位 (相对每个池的排名和超额)
  → 板块轮动 (池与池之间的强弱对比)
  → 信号生成 (35+ 标签，5 大类)
  → 结论输出 (Beta/Alpha/风险/摘要)
  → 历史分析 (365天回溯)
  → Dashboard JSON (前端消费)
```

四步管道，不可跳步：`price.run → dailyreport.run → build_history_analysis → build_dashboard_view`

## 模块卡片

### M01. 锚定标的

| 项目 | 内容 |
|------|------|
| **解决什么问题** | 分析必须围绕一个中心，否则"联动"无从谈起 |
| **输入** | 一只 A 股代码（当前：688333.SH 铂力特） |
| **输出** | 标的身份、所属行业、产业链位置 |
| **设计决策** | 锚定标的不进入自己的 benchmark 计算 — 你不能跟自己比"相对强弱"，否则永远没有偏离。锚定的是"参照系原点"，不是"参与者" |
| **为什么不用行业指数** | 宽泛的行业指数（如军工指数）包含大量与铂力特无关的股票，噪声太大。需要手工定义有业务关联的池子 |

---

### M02. 四池体系 (Universe)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "板块"不是一个东西。铂力特同时受产业链传导、主题情绪、短期交易资金影响，混在一起看会得出错误的结论 |
| **输入** | config/pools.yaml 定义的 4 个 Universe |
| **输出** | 每个池的每日状态（中位数收益、上涨比、成交额倍数） |

**四个池各回答不同的问题：**

| 池 | 名称 | 回答的问题 | 能否做 benchmark |
|----|------|-----------|-----------------|
| direct_peers | 增材制造本业确认池 | 同行怎么样？基本面驱动的可比公司 | 是 |
| industry_chain | 商业航天硬科技主池 | 产业链传导到哪了？上下游的真实景气 | 是 |
| theme_pool | 商业航天主题温度计 | 市场情绪在炒什么？投机资金的方向 | 否 |
| trading_watchlist | 交易联动与风险映射池 | 短期资金在干嘛？高 Beta 标的的领先信号 | 否 |

**关键设计决策：**

1. **主题池和交易观察池不能做 benchmark，但必须计算状态。** 原因：这两个池代表的是情绪和资金，不是基本面参照。但如果你不知道主题池在升温，就理解不了"为什么主线池没动但个股在涨"。

2. **一只股票可以同时属于多个池，角色不同。** 比如华曙高科在 direct_peers 里是"核心可比"，在 theme_pool 里是"情绪代理" — 同一个价格数据，在不同池里解读方式不同。

3. **为什么是 4 个池而不是更多？** 每多一个池，分析复杂度指数增长（4 池已有 6 对两两比较）。4 个池对应 4 个独立维度：基本面、产业链、情绪、资金，覆盖了联动分析的主要信息来源。

### M03. 池状态计算 (Pool State)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | 每个池今天整体是强还是弱？需要量化，不能凭感觉 |
| **输入** | 池内所有成分股的当日行情数据 |
| **输出** | 中位数收益 (median_return)、上涨比 (up_ratio)、成交额倍数 (volume_multiplier) |

**为什么用中位数而不是均值？** A 股小池子（5-10 只票）里，一只涨停或跌停就会严重拉偏均值。中位数对极端值不敏感，更能代表"池子的整体感觉"。

**为什么看三个指标而不是只看收益？**
- 中位数收益 → 方向（涨还是跌）
- 上涨比 → 共识度（是普涨还是分化）
- 成交额倍数 → 力度（是真金白银还是无量阴跌）

三者组合才能区分"温和普涨"和"放量暴涨"这种质的不同。

### M04. 个股定位 (Anchor Position)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | 铂力特在各个池子里排第几？是领涨还是拖后腿？ |
| **输入** | 锚定标的当日收益 + 每个池的成分股收益排名 |
| **输出** | 每个池内的排名百分位、相对强度 (relative_strength) |

**relative_strength 的含义：** 个股收益减去池中位数收益。>0.5% 算"个股偏强"，<-0.5% 算"个股偏弱"。这个 0.5% 的阈值来自 A 股日收益的典型波动 — 太小（0.1%）会把噪声当信号，太大（2%）会漏掉有意义的偏离。

**为什么要在每个池里分别定位？** 同一天，铂力特可能跑赢产业链池但跑输主题池 — 这说明基本面在支撑但情绪在退潮，是两种完全不同的市场含义。

---

### M05. 板块轮动 (Group Rotation)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | 四个池子之间，谁在领涨？资金在往哪个方向流？ |
| **输入** | 四个池的中位数收益 |
| **输出** | 最强池、最弱池、池间强弱差 |

**核心逻辑：** 池子之间的相对强弱反映了市场资金的偏好迁移：
- 主线池强于主题池 → 基本面驱动，持续性可能更好
- 主题池强于主线池 → 情绪驱动，需要警惕持续性
- 交易观察池升温 → 短期资金在进场，可能是波动前兆

**为什么不用相关系数？** 相关系数衡量的是"同步性"，不是"谁领先"。我们需要的是"谁比谁强"的相对关系，所以直接用中位数之差，简单直观。

### M06. 信号系统 (Signals)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | 把池状态和个股定位的数值结论，翻译成可读的标签。原始数据（中位数 0.3%、上涨比 60%）对交易者没有直接意义，标签（"行业Beta为正"）才有 |
| **输入** | 池状态、个股定位、板块轮动的数值结果 |
| **输出** | 35+ 信号标签，每个带 Evidence（来源池、字段、阈值、实际值） |

**5 大信号类别：**

| 类别 | 信号举例 | 业务含义 |
|------|---------|---------|
| Beta（行业环境） | 行业Beta为正/负、行业扩散增强/不足、行业分化 | 板块整体方向和内部一致性 |
| Alpha（个股Alpha） | 个股Alpha为正/负、跑赢/跑输主线池、处于行业前排/后排 | 个股相对板块的独立强弱 |
| Volume（资金成交） | 放量上涨/下跌、缩量调整、放量滞涨、资金价格共振/背离 | 量价关系和资金行为 |
| Rotation（板块轮动） | 主线池强于主题情绪、交易观察池升温/降温 | 池子之间的资金流向 |
| Abnormal（联动背离） | 行业强但个股弱、主题情绪强但主线池弱 | 方向矛盾 — 通常是最值得关注的信号 |

**关键设计决策：信号必须带 Evidence。** 每个标签生成时记录来源池、源字段、阈值和实际值。这保证了从结论可以回溯到数据，不是黑箱标签。一个标签"行业Beta为正"背后是 `industry_chain.median_return = 1.2% > threshold 0.5%`。

**阈值从哪来？** Beta/Alpha 的 ±0.5% 阈值参考了 Jegadeesh & Titman (1993) 动量研究的日频分段思路，以及 IBD (Investor's Business Daily) 的相对强度评级中对 0.5% 日超额的划分。但当前阈值未经本数据集回测验证，是先验设定。

### M07. 结论构建 (Conclusion Builder)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | 35+ 信号标签太多，交易者需要一个一句话结论 |
| **输入** | 信号系统的输出 + 池状态 |
| **输出** | industry_beta (positive/neutral/negative)、anchor_alpha (同)、risk_level (low/medium/high)、summary (叙事文本)、next_watch (关注点列表) |

**industry_beta 的判定：** 从 Beta 类信号中找"行业Beta为正/负"标签，有则定，无则中性。本质是：产业链主池中位数涨了就是正，跌了就是负，小涨小跌就是中性。

**anchor_alpha 的判定：** 从 Alpha 类信号中找标签。除了"个股Alpha为正/负"，还认"跑赢/跑输主线池" — 因为在某些情况下 Alpha 信号没有直接生成，但相对强弱信号能替代。

**risk_level 的判定优先级：**
1. 数据不足 → high（连数据都没有，结论不能信）
2. 有 Abnormal 类信号 → high（方向矛盾意味着市场在分歧，风险最大）
3. 数据部分缺失 → medium
4. 池内分化严重（3+ 强或 3+ 弱）→ medium
5. 以上都没有 → low

**为什么风险等级和数据质量绑定？** 因为这个系统的核心假设是"池状态能代表板块"。如果数据缺失，这个假设就不成立，所有下游结论都不可信。所以数据质量本身就是最大的风险来源。

### M08. 五维状态画像

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "今天是怎样的一个日子？" — 用 5 个维度把每天的市场处境编码成一个可比较的向量 |
| **输入** | 结论构建的 3 个维度 + 板块轮动的 2 个维度 |
| **输出** | industry_beta, anchor_alpha, risk_level, strongest_group, weakest_group |

**5 个维度从哪来？**

不是凭空选的，也不是从外部框架抄的。它们是系统 8 层架构的自然产物 — L4 到 L7 每一层算完东西后都需要一个一句话结论，这些结论合起来就是这 5 个维度：

| 架构层 | 这一层的任务 | 自然产出的结论 | 对应维度 |
|--------|------------|--------------|---------|
| L4 池状态计算 | 算每个池今天整体强还是弱 | 板块涨了还是跌了 | industry_beta |
| L5 个股定位 | 算个股在每个池里的相对位置 | 个股跑赢还是跑输 | anchor_alpha |
| L6 板块轮动 | 比较池与池之间的强弱 | 谁最强？谁最弱？ | strongest_group + weakest_group |
| L7 信号 + 数据质量 | 生成信号标签 + 检查数据完整性 | 结论能不能信？ | risk_level |

反过来说：如果架构里没有 L6（板块轮动），就不会有 strongest/weakest_group。如果 L7 不关心数据质量，就不会有 risk_level。维度跟着架构走，不是先定维度再建架构。

**5 个维度各自回答什么：**

| 维度 | 回答的问题 | 为什么重要 |
|------|-----------|-----------|
| industry_beta | 产业链今天强不强？ | 决定大环境：产业链行情还是产业链不行 |
| anchor_alpha | 铂力特自己强不强？ | 决定个股有没有独立逻辑，还是只是板块的影子 |
| risk_level | 今天的结论能不能信？ | 数据不好或方向矛盾时，所有参考价值打折 |
| strongest_group | 产业链里谁在领涨？ | 今天是哪个细分方向在带节奏 |
| weakest_group | 产业链里谁在拖后腿？ | 哪个方向在被抛售 |

**为什么是这 5 个而不是别的？** 因为锚定联动分析的核心场景是"找处境相似的历史日"。两个交易日如果产业链环境相同（beta）、个股强弱相同（alpha）、数据可信度相同（risk）、内部结构接近（最强/最弱池），那它们的市场"处境"就是相似的 — 这就是今日历史映射的检索基础。

**为什么不用更多维度？** 维度越多，每个维度能取的值越少（否则组合爆炸），可比较的历史样本就越少。5 个维度 × 各 3 个取值 ≈ 243 种状态，在 365 天样本里每种状态平均 1-2 个样本，已经偏少。加维度会让样本更稀疏。

### M09. 今日历史映射 (History Mapping)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "今天这样的日子，历史上之后怎么走的？" — 用历史路径给今天提供参考，但不做预测 |
| **输入** | 目标日的五维状态 + 信号标签集合；全部历史样本 |
| **输出** | 相似案例 Top 5、路径统计 (T+1/3/5 收益/胜率/超额)、路径标签 |

**算法：状态-信号联合相似度 kNN**

```
相似度 = 状态相似 × 0.6 + 信号 Jaccard × 0.4
```

- **状态相似 (0.6 权重)：** 五维状态的加权比较。industry_beta 和 anchor_alpha 各 0.25（最核心），risk_level 0.20，strongest/weakest_group 各 0.15。序数型字段（beta/alpha/risk）用相邻评分（相同=1，相邻=0.5，远=0），名义型字段（group）用精确匹配。
- **信号 Jaccard (0.4 权重)：** 两天的信号标签集合的交集/并集比。衡量"这两天触发了多少相同的信号"。

**为什么状态占 0.6、信号占 0.4？** 状态维度是结构性的（大盘环境+个股定位），决定了大方向；信号是细节性的（具体什么信号触发了），在状态相似的基础上做微调。如果反过来给信号更高权重，两天可能因为触发了相同的小概率信号而"看起来相似"，但市场环境其实完全不同。

**自适应 Top N：** `min(12, max(5, round(候选数 × 0.15)))`。太少（固定 5 个）会漏掉相似样本，太多（全部）会引入噪声。15% 的比例在 365 天样本里大约选出 50 个候选中的 7-8 个。

**路径标签（规则引擎）：** 从 Top N 样本的 T+1/3/5 统计推导，6 档：

| 标签 | 条件 | 含义 |
|------|------|------|
| 强势延续 | T+1/3/5 收益全正 + 胜率全 ≥55% | 三个窗口都偏多 |
| 冲高回落 | T+1 涨 >0.5% 但 T+1 与 T+5 差 ≥0.7% | 短期冲高但撑不住 |
| 继续走弱 | 2+ 窗口收益 < -0.3% | 持续偏空 |
| 弱势修复 | T+1 或 T+3 >0.3% 但 T+5 不正或胜率 <50% | 短期反弹但持续性存疑 |
| 分化震荡 | 所有窗口收益绝对值 <0.2% 且胜率 45-55% | 没有明确方向 |
| 样本分歧 | 以上都不满足 | 样本内部矛盾大 |

**为什么不直接展示相似度分数？** 相似度是算法内部度量，对交易者没有直觉意义。路径标签把数值结论翻译成"这组历史案例整体在走什么路"，更容易形成判断。

### M10. 状态迁移路径 (Transition Heatmap)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "今天这个状态，明天最可能变成什么状态？" — 不是看相似历史日，而是看状态本身的转移概率 |
| **输入** | 365 天的 (industry_beta + anchor_alpha) 序列 |
| **输出** | 9×9 转移矩阵、Top 3 迁移路径、关注点 |

**九宫格状态定义：** industry_beta (3) × anchor_alpha (3) = 9 种状态，格式为"产业链+个股"（产业链强+个股强、...产业链弱+个股弱）。

**为什么只用 beta+alpha 做九宫格，不用全部 5 维？** 5 维会产生 3×3×3×N×M 的超大矩阵，365 天的样本量完全不够填充。beta+alpha 是最核心的 2 个维度，9 种状态每种平均有 40 天样本，统计上有意义。

**矩阵读法：** 橙色行是当前状态，每格数字是从当前状态转移到对应状态的频率。40% 以上标强色，20-40% 标中色，<20% 标弱色。

**Top 3 路径的价值：** 矩阵太密，交易者不会逐格看。Top 3 直接告诉"最可能的 3 个方向 + 概率 + 历史样本数 + T+1/3 收益"，快速形成预期。

### M11. 近期稳定性 (Stability Panel)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "最近铂力特是持续跑赢航天主池，还是靠一两天运气？" — 区分趋势和波动 |
| **输入** | 滚动 5/10 日超额收益序列、每日个股/池/超额序列 |
| **输出** | 当前稳定性结论、5 日/10 日超额趋势、今日偏离程度、2 张趋势图 |

**三个指标各回答什么：**
- 5 日超额 → 短期动量：最近一周有没有持续跑赢航天主池？
- 10 日超额 → 中期趋势：最近两周呢？
- 今日偏离 → 今天的超额是延续趋势还是突然偏离？

**稳定性判定逻辑：** 如果 5/10 日超额持续抬升，且今日没有异常偏离，标记"稳定"；如果超额在震荡或掉头，标记"变化中"。

**为什么"稳定性"重要？** 因为它决定了你是否可以依赖"历史规律"。如果个股超额在持续走强（稳定跑赢航天主池），那历史映射的路径参考更有意义；如果超额在震荡（跑赢跑输交替），历史路径的参考价值就打折 — 因为今天的"超额"可能明天就消失了。

### M12. 历史性格档案 (Personality Profile)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "这只股票长期是什么性格？" — 不是看某一天，而是看整个样本周期的行为模式 |
| **输入** | 365 天全部样本的信号标签和收益数据 |
| **输出** | 性格分布环形图、偏好/规避/反直觉/陷阱四类模式、度量条、产业联动关系 |

**四类行为模式：**

| 类型 | 含义 | 业务价值 |
|------|------|---------|
| 偏好环境 | 某信号出现时，个股后续收益显著为正 | 知道什么条件下这只票容易涨 |
| 规避环境 | 某信号出现时，个股后续收益显著为负 | 知道什么条件下应该回避 |
| 反直觉 | 直觉上应该是正面/负面，实际相反 | 最有价值的信号 — 市场预期错了 |
| 陷阱 | 看似正面信号，实际后续收益为负 | 防止追高被套 |

**为什么"反直觉"和"陷阱"单独列出？** 传统分析只看"什么条件下涨/跌"，但最有交易价值的是"市场普遍以为会涨但实际会跌"的场景。这些是超额收益的来源 — 因为大多数人会做错方向。

**产业联动关系：** 锚定标的与四个池的 sameDayCorr (同日相关系数) + avgRelativeStrength (平均相对强度) + relation (强跟随/弱跟随/独立/反向)。这回答"铂力特到底在跟谁走"。

### M13. 信号时间轴 (Signal Timeline)

| 项目 | 内容 |
|------|------|
| **解决什么问题** | "过去的信号怎么演变到今天的？" — 不是看某一天的快照，而是看信号的历史变迁 |
| **输入** | 365 天的信号标签序列 + 价格序列 |
| **输出** | 价格折线图 + 信号密度标记、4 类信号轨道、信号变迁统计 |

**4 类信号轨道：** 偏好（红）、规避（绿）、反直觉（紫）、陷阱（橙）。每条轨道上，信号出现的日子点亮，没出现的留空。

**为什么需要轨道视图？** 折线图只能看价格，看不到信号。把信号拆成轨道后，可以直观看到"偏好信号在最近 20 天密集出现但陷阱信号也在增加" — 这种模式在纯价格图上完全看不到。

**信号变迁：** 对比早期（前 70%）和近期（后 20 天）的高频信号。如果早期主导是"偏好环境"但近期变成了"规避环境"，说明市场状态在切换。
