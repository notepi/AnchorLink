# 股东人数量化策略分析系统 - 技术文档

## 一、项目概述

本项目基于**股东人数变化**与**股东人数分位数**的双重维度，构建了一套完整的量化交易策略分析系统。通过对华英农业(002321)的历史数据进行回测，寻找最优的买卖时机组合。

### 核心逻辑

**买入信号**: 当股东人数增长率低于设定阈值，且当前股东人数分位数低于设定阈值时，触发买入。

**卖出策略**: 固定持有20个交易日后卖出，记录三种卖出结果（最高价/最低价/中位数）。

## 三、核心指标计算

### 3.1 股东人数增长率 (Growth Rate)

**公式**:
```
增长率 = (本次股东人数 - 上次股东人数) / 上次股东人数 × 100%
```

**含义**:
- 负值表示股东人数减少（筹码集中）
- 正值表示股东人数增加（筹码分散）

**使用方式**: 原始数据直接提供，需除以100转换为小数形式。

### 3.2 股东人数分位数 (Percentile)

**预计算常量表** (基于历史数据统计):
```python
SHAREHOLDER_PERCENTILES = {
    10: 34112,   # 10分位：股东人数 ≤ 34112
    20: 37263,   # 20分位：股东人数 ≤ 37263
    30: 39385,   # 30分位
    40: 42349,   # 40分位
    50: 46076,   # 中位数
    60: 49884,   # 60分位
    70: 57243,   # 70分位
    80: 65043,   # 80分位
}
```

**计算逻辑**:
```python
def get_percentile(shareholders):
    """根据股东人数返回对应的分位数"""
    sorted_percentiles = sorted(SHAREHOLDER_PERCENTILES.items(), key=lambda x: x[1])
    for quantile, threshold in sorted_percentiles:
        if shareholders <= threshold:
            return quantile  # 返回分位数值 (10, 20, 30...)
    return 80  # 超过所有阈值返回最高分位
```

**含义**:
- 低分位 (10-30): 股东人数处于历史低位，筹码高度集中
- 高分位 (60-80): 股东人数处于历史高位，筹码分散

## 五、买卖信号逻辑

### 5.1 买入条件

```python
def check_buy_conditions():
    # 条件1: 增长率条件
    # 增长率 <= 增长率阈值 (负值越大，条件越严格)
    growth_condition = growth_rate <= growth_threshold

    # 条件2: 分位数条件
    # 当前分位数 <= 分位数阈值 (阈值越低，条件越严格)
    quantile_condition = current_percentile <= quantile_threshold * 100

    # 两个条件同时满足才买入
    if growth_condition and quantile_condition:
        execute_buy()
```

**策略解读**:
| 参数组合 | 含义 |
|----------|------|
| G=-5%, Q=20% | 股东暴减5%以上 且 分位处于低位(≤20%) |
| G=-2%, Q=40% | 股东减少2%以上 且 分位处于中低位(≤40%) |
| G=+3%, Q=60% | 股东增加不超过3% 且 分位不超60% |

### 5.2 卖出条件

```python
def check_sell_conditions():
    # 固定持有20个交易日后卖出
    if hold_days >= 20:
        execute_sell()
```

### 5.3 三种卖出结果记录

每次交易记录三种假设性卖出结果：

| 卖出类型 | 价格 | 收益率 | 含义 |
|----------|------|--------|------|
| 卖出-最高价 | 持有期最高价 | (最高价-买入价)/买入价 | 理想最优收益 |
| 卖出-最低价 | 持有期最低价 | (最低价-买入价)/买入价 | 最差情况 (MAE) |
| 卖出-中位数 | 持有期价格中位数 | (中位数-买入价)/买入价 | 平均收益 |

## 六、风险收益指标计算

### 6.1 MFE (Maximum Favorable Excursion)

**定义**: 最大有利偏移，即持有期间的最高潜在收益。

**计算公式**:
```
MFE = (持有期最高价 - 买入价) / 买入价
```

**含义**: 如果能在最高点卖出，能获得的最大收益。

### 6.2 MAE (Maximum Adverse Excursion)

**定义**: 最大不利偏移，即持有期间的最大浮亏。

**计算公式**:
```
MAE = (持有期最低价 - 买入价) / 买入价
```

**含义**: 持有期间可能遭遇的最大亏损，代表风险暴露。

### 6.3 RR Ratio (风险收益比)

**计算公式**:
```
RR_Ratio = MFE_Mean / |MAE_Mean|
```

**解读**:
| RR值 | 含义 |
|------|------|
| RR > 2.0 | 优秀：潜在收益是风险的2倍以上 |
| RR 1.5-2.0 | 良好：收益风险比合理 |
| RR 1.0-1.5 | 一般：收益勉强覆盖风险 |
| RR < 1.0 | 较差：风险大于收益 |

### 6.4 胜率 (Win Rate)

**计算公式**:
```
Win_Rate = 盈利交易次数 / 总交易次数
```

### 6.5 利润留存率 (Retention)

**计算公式**:
```
Retention = 最终收益率 / MFE
```

**含义**: 衡量"过山车"程度。留存率低说明虽然有高点，但没能守住利润。

## 八、置信度评分模型 (V5终极版)

### 8.1 评分公式

**买入区评分 (RR > 1.0)**:
```
Score_V5 = [(RR - 1.0) × 20.0 - Efficiency_Penalty] × log10(Trades + 1)
```

其中:
- `RR_Advantage_Score = (RR - 1.0) × 20.0`
- `Efficiency_Penalty`: 当 1.0 < RR ≤ 1.2 时，惩罚 = RR_Advantage × 0.5
- `Reliability_Multiplier = log10(Trades + 1)`

**卖出区评分 (RR ≤ 1.0)**:
```
Score_V5 = (RR - 1.0) × 10.0
```
当 RR ≤ 0.05 时，Score = -20.0 (极限风险)

### 8.2 操作信号映射

| 评分范围 | 信号等级 | 操作建议 |
|----------|----------|----------|
| ≥ 15.0 | S1 | 强力买入 (效率/确定性最优) |
| 5.0 ~ 15.0 | S2 | 核心买入 (高效率且稳健) |
| 0 ~ 5.0 | S3 | 临界买入/观察 |
| -2.0 ~ 0 | M1 | 临界风险/不买入 |
| -10.0 ~ -2.0 | M2 | 结构性卖出/规避 |
| < -10.0 | M3 | 强制卖出/空仓 (极限风险) |

### 8.3 评分逻辑解读

1. **RR优势得分**: 基础分，反映收益风险比
2. **效率惩罚**: 防止RR略大于1的"垃圾策略"得分过高
3. **可靠性乘数**: 样本量越大，评分越可靠（log10平滑）

## 九、数据处理流程

### 9.1 数据对齐

**问题**: 股东数据是定期公告（非每日），股价数据是每日数据。

**解决方案**:
```python
# 使用前向填充(Forward Fill)
shareholder_data = shareholder_data.reindex(price_dates, method='ffill')
```

**效果**: 公告日之后的所有交易日都使用该公告数据，直到下一个公告发布。

### 9.2 非交易日处理

**问题**: 股东公告可能在周末/节假日发布。

**解决方案**: 通过 `reindex + ffill` 自动顺延到下一个交易日生效。

### 9.3 重复数据处理

**问题**: 同一天可能发布多条股东数据。

**解决方案**:
```python
# 按公告日和统计日排序，保留最新的统计日数据
shareholder_data.sort_values(by=['date', 'stat_date'], ascending=[True, True])
shareholder_data.drop_duplicates(subset=['date'], keep='last')
```

### 15.3 三个关键问题的精确答案

#### Q1: Ret_Mean 到底是哪种退出？

**答案: 卖出-中位数的收益率**

代码证据 (`analyze_shareholders_V2.py` 第135-140行):
```python
# 3. 提取最终结局 (优先取中位数)
median_row = group[group['操作'] == '卖出-中位数']
if median_row.empty: median_row = sell_rows.iloc[0]
else: median_row = median_row.iloc[0]

final_return = median_row['收益率']  # ← 这就是 Ret_Mean 的来源
```

**为什么不是MFE？** MFE是"最大潜在收益"，是理想化指标，用于评估进攻潜力。
**为什么不是MAE？** MAE是"最大浮亏"，是风险指标，用于计算RR Ratio。
**为什么选中位数？** 代表"典型持有结果"，是可实现的收益预期。

#### Q2: 510组参数怎么映射到35格？

**答案: 不是参数映射，而是交易结果按实际值分档**

关键区别：
| 概念 | 说明 |
|------|------|
| **阈值参数** | 触发条件，如 growth_threshold=-5%, quantile_threshold=20% |
| **实际值** | 触发时真实发生的增长率/分位数，如 -7.2%, 15% |

分档逻辑 (`analyze_shareholders_V2.py` 第168-175行):
```python
# 按实际发生的增长率分档
growth_bins = [-np.inf, -0.05, -0.025, -0.015, 0.0, 0.01, 0.03, np.inf]
trade_df['G_Bin'] = pd.cut(trade_df['Growth'], bins=growth_bins, ...)

# 按实际发生的分位数分档  
quantile_bins = [0, 20, 40, 55, 70, 100]
trade_df['Q_Bin'] = pd.cut(trade_df['Quantile'], bins=quantile_bins, ...)
```

**映射流程图**:
```
510组参数 → 各自产生信号 → 触发时记录实际值 → 按实际值落入G/Q格子
   ↓
例如: 参数 threshold=-5% 触发时实际 growth=-7.2%
     → 落入 G1(暴减<-5%) 格子
     
例如: 参数 threshold=20% 触发时实际 quantile=15%
     → 落入 Q1(低位<20%) 格子
```

**本质**: G_Bin/Q_Bin 是对"所有信号池"的分类统计，不是对"参数组合"的映射。

#### Q3: 多股票还是单股票？Count为什么这么大？

**答案: 单股票，但多参数组合汇总**

**Count来源分解**:
```
G1×Q1 的 Count=7203 来自:
├── 510组参数中，所有满足 G_阈值≤-5% 且 Q_阈值≤20% 的组合
├── 每组参数在12年回测期内产生的交易
└── 这些交易触发时的实际增长率<-5% 且 实际分位数<20%

具体计算:
- 参数组合数: 约 21个G阈值 × 2个Q阈值 = 42组参数会贡献到G1×Q1
- 每组参数平均产生: 7203 / 42 ≈ 171 笔交易
- 每年每组参数平均: 171 / 12 ≈ 14 笔交易/年
```

**为什么一天可能多笔交易？**
- 不同参数组合可能在同一天触发
- 例如 threshold=-5% 和 threshold=-6% 可能在同一天都满足条件
- 但代码限制了**同一天同一参数组合只能买入一次**

**Count计算代码** (`analyze_shareholders_V2.py` 第184行):
```python
stats = trade_df.groupby(['G_Bin', 'Q_Bin']).agg({
    'Growth': 'count'  # ← 每笔交易算1次
}).rename(columns={'Growth': 'Count'})
```

**纠正之前的错误**:
- ❌ Count/3 = 实际交易数 (错误)
- ✅ Count = 实际交易笔数 (正确，因为汇总时每笔交易只计1次)

#### Q2: 中位数卖出价格的精确定义

**定义**: 持有期窗口内**所有收盘价的统计中位数**

**计算步骤** (代码 `shareholder_strategyV3_F.py` 第230-233行):
```python
# Step 1: 计算统计中位数
median_price = np.median(prices)  # 如 [3.0, 3.1, 2.9, ...] → 3.05

# Step 2: 找到最接近该值实际价格（从后往前）
price_diffs = np.abs(np.array(prices) - median_price)
median_index = len(price_diffs) - 1 - np.argmin(price_diffs[::-1])

# Step 3: 使用该价格对应的原始日期
median_date = dates[median_index]
```

**关键区别**:
| 概念 | 定义 | 说明 |
|------|------|------|
| **统计中位数价格** | `np.median(prices)` | 所有价格的50%分位值 |
| **中位数卖出日期** | 最接近该价格的实际日期 | 可能是第1天到第N天 |

**示例**:
```
持有期价格序列: [3.0, 3.2, 3.1, 2.9, 3.3, 3.4, 3.0, ...]
统计中位数: 3.1
最接近3.1的实际价格: 3.1（第3天）
→ 中位数卖出日期 = 第3天
→ Avg_Days 贡献值 = 3
```

**特殊情况**:
- 如果价格单边上涨，中位数价格可能在第10天左右出现
- 如果价格单边下跌，中位数价格可能在第10天左右出现
- 如果价格剧烈波动，中位数价格可能在任何一天出现

**Avg_Days 的含义**:
```
Avg_Days = mean(所有交易的"中位数价格出现天数")
```
它反映的是"典型价格出现的平均时点"，而非"固定持有期"。

## 十七、指标迁移指南：从旧方案到新方案

### 17.1 核心差距分析

|----------|------|-------------|------|
| **Ret_Mean** | 中位数价格退出收益 | 无（只有 Oracle 最高/最低） | ⚠️ **需添加** `fwd_20d_median_return` |
| MFE_Mean | 最高价退出收益 | `upsideMean` | ✅ 已有 |
| MAE_Mean | 最低价退出收益 | `adverseMean` | ✅ 已有 |
| RR Ratio | MFE / \|MAE\| | 已有 | ✅ 已有 |
| Retention | 最终收益 / MFE | 无 | ⚠️ 可添加 |
| Profit_Factor | 盈亏比 | 无 | ⚠️ 可添加 |
| VaR_95 | 收益5%分位数 | 无 | ⚠️ 可添加 |

### 17.2 中位数退出收益的定义与计算

**定义**：
```
fwd_20d_median_return = (中位数价格 - 买入价) / 买入价
```

其中"中位数价格"是持有期内所有收盘价的统计中位数，而非排序后中间那条记录。

**计算步骤**：

```python
# Step 1: 计算统计中位数价格
median_price = np.median(prices)  # prices = [p1, p2, ..., p20]

# Step 2: 找到最接近该值的实际价格（从后往前，保持一致性）
price_diffs = np.abs(prices - median_price)
median_index = len(price_diffs) - 1 - np.argmin(price_diffs[::-1])

# Step 3: 计算收益
median_return = (median_price - entry_price) / entry_price
median_day = median_index + 1  # 该价格出现的天数
```

**为什么比 Oracle 更有价值**：

| 对比维度 | Oracle (MFE) | 中位数收益 | 说明 |
|----------|-------------|-----------|------|
| 可实现性 | 几乎不可能 | 大概率可实现 | 中位数是典型价格 |
| 信息量 | 只反映上沿 | 反映整体分布 | 中位数代表中心趋势 |
| 风险考量 | 无 | 隐含波动信息 | 中位数偏离买入价说明波动 |

### 17.3 完整指标计算口径

**空间层指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| `fwd_max_return` | `(max(prices) - entry) / entry` | 最高价收益 (MFE) |
| `fwd_min_return` | `(min(prices) - entry) / entry` | 最低价收益 (MAE) |
| `fwd_rr_ratio` | `fwd_max_return / abs(fwd_min_return)` | 风险收益比 |

**常态表现层指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| `fwd_median_return` | `(median(prices) - entry) / entry` | **核心收益指标** |
| `fwd_median_day` | 中位数价格出现的天数 | 典型价格时点 |
| `fwd_return_var95` | `percentile(returns, 5%)` | 最差5%情况 |

**路径层指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| `fwd_peak_day` | `argmax(returns) + 1` | 达峰天数 |
| `fwd_trough_day` | `argmin(returns) + 1` | 达谷天数 |
| `fwd_retention` | `fwd_median_return / fwd_max_return` | 利润留存率 |

**风险层指标**：

| 指标 | 公式 | 含义 |
|------|------|------|
| `win_rate` | `count(ret>0) / n` | 胜率 |
| `profit_factor` | `sum(盈利) / abs(sum(亏损))` | 盈亏比 |
| `var_95` | `percentile(returns, 5%)` | 风险价值 |


### 17.5 新旧方案 PF 差异原因

| 方案 | PF 输入序列 | 特征 | 结果 |
|------|------------|------|------|
| **旧方案** | 中位数收益 | 有正有负 | 具有区分度，PF ∈ [0, ∞) |
| **当前方案** | Oracle 上行空间 | 通常为正 | 全是极端值，PF ≈ 9.90 |

**解决方案**：
1. 使用 `fwd_median_return` 序列计算 PF
2. 或者使用 `fwd_固定天数_return` (如 T+20 收益) 计算 PF

### 17.6 迁移实施要点

**不迁移的部分**：
- ❌ 参数遍历产生的重复交易池
- ❌ Count 的累加方式（同一天多参数重复计数）
- ❌ 信号触发的多参数叠加逻辑

**需要迁移的部分**：
- ✅ 中位数退出收益 (`fwd_median_return`) 作为核心收益指标
- ✅ Retention 用于识别"过山车"行情
- ✅ 基于中位数收益计算 PF 和 VaR
- ✅ 达峰/达谷天数用于路径分析

**迁移代码示例**：

```python
def calc_forward_metrics(prices: np.ndarray, entry_price: float) -> dict:
    """计算完整前向指标"""
    if len(prices) == 0 or entry_price <= 0:
        return {}
    
    returns = (prices - entry_price) / entry_price
    
    # 空间层
    max_ret, min_ret = np.max(returns), np.min(returns)
    
    # 常态层
    median_price = np.median(prices)
    median_ret = (median_price - entry_price) / entry_price
    
    # 路径层
    peak_day = np.argmax(returns) + 1
    trough_day = np.argmin(returns) + 1
    retention = median_ret / max_ret if max_ret > 0 else 0
    
    return {
        'fwd_max_return': max_ret,        # MFE
        'fwd_min_return': min_ret,        # MAE
        'fwd_rr_ratio': max_ret / abs(min_ret) if min_ret != 0 else np.inf,
        'fwd_median_return': median_ret,  # ⭐ 核心指标
        'fwd_peak_day': peak_day,
        'fwd_trough_day': trough_day,
        'fwd_retention': retention,
    }
```

## 十六、已知问题与改进建议

### 16.1 当前模型的局限性

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 分位数全样本泄漏 | 回测收益偏乐观 | ⚠️ 高 |
| 公告日收盘价入场 | 存在未来函数 | ⚠️ 中 |
| 未包含印花税 | 收益偏高估 | ⚠️ 低 |
| 固定持有期 | 不适应市场变化 | ℹ️ 设计选择 |

### 16.2 改进建议

1. **分位数滚动计算**: 仅用当前时点之前的历史数据

2. **入场时机调整**: 改为公告日次日开盘价

3. **动态持有期**: 根据波动率调整持有天数

4. **止盈止损**: 添加动态止盈止损条件

5. **多股票验证**: 在其他股票上验证策略有效性
