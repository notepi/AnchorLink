## 总结：哪些问题答得好，哪些缺数据

### 关键洞察


1. **Q1（归因）现有数据已经能答 90%**——所有需要的字段都在 `industry_snapshot.json` 和 `history_signal_lift.csv` 里。问题不是缺数据，是 UI 没把它们组合起来。

2. **Q2（明日预期）现有数据能答 85%**——`history_summary.csv` 有完整的 next_1d/3d/5d，可以算分位数和胜率。**但用户感觉的"准"在 242 天数据里没有显著支撑**：当前象限"行业弱+个股强" 在 9 象限里 T+1 排名靠后（详见 2.4 和 2.5）。用户的"准感"可能来自：
   - **样本偏差**（最近这几次刚好对了）
   - **方向预测的微弱信号**（即使胜率 48%，比 50% 略低，但他记得对的多）
   - **置信区间的解释力**（HistoryMapping 给的 CI 让人感觉有依据）

3. **Q3（信号有效性）96→242 天对比有意义**——`放量上涨` 仍然是 trap，但部分反直觉信号在 242 天样本下数值发生了变化。**建议把 lift 数字直接挂在 currentMapping 标签旁**（如 "行业弱但个股强 → 历史 T+1 -0.5%/胜率 42%"），让用户看到"信号本身的代价"。

4. **Q4（联动）现有数据能答 80%**——最近 60 天 industry_chain 相关性 0.61，比预期低。**铂力特已经进入分化型联动**（跑赢比例 65%，相关性 < 0.5），这是为什么 Q1 里 Alpha 占主导。

### 给 UI 的建议（不写代码，只是结论）


**优先做**（基于现有数据，零新增分析）：
1. 顶部加"今日归因"区块：直接展示 1.1 的拆解 + 1.2 的信号 lift 表
2. HistoryMapping 升级：把 T+1 平均值改成"分布（中位数+P10/P25/P75/P90+胜率）"
3. 每个 currentMapping 标签后挂"历史 T+1 = X% / 胜率 Y%"

1. ~~新增 PoolToPoolLinkage（池间 lead-lag）~~——peak lag 已经看出来基本同步，没有显著结构性领先关系
2. ~~新增 dispersion / concentration~~——现有 9 象限和信号体系已经覆盖了用户的 4 个问题
3. ~~继续完善 PersonalityProfile~~——用户明确说看不懂，应该简化或隐藏

**用户原始问题"如果我要做铂力特，更行业联动的分析"的答案**：
