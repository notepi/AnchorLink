# 创建业务评审 PRD

## Context

用户要求创建一个面向业务专家的 PRD 文档，用于评估方案设计是否合理。

**与现有 PRD 的区别**：
- 现有 `docs/prd.md`：面向开发团队，侧重技术实现
- 新 PRD：面向业务专家，侧重方案合理性评估

---

## 1. 为什么要做这个系统

### 1.1 业务痛点

投资者跟踪铂力特（688333.SH）时面临：

| 痛点 | 具体问题 |
|------|----------|
| 信息分散 | 行情、资金、新闻数据分散在多个平台 |
| 缺乏参照 | 不知道铂力特在商业航天板块中的相对位置 |
| 主力难跟踪 | 资金流向数据专业性强，难以快速理解 |
| 连续性缺失 | 只有当日数据，无法判断趋势连续性 |

### 1.2 解决目标

将"每天花30分钟看盘+复盘"压缩为"5分钟看一份报告"。

---

## 2. 为什么这么设计

### 2.1 为什么选择"相对强弱"而非绝对涨跌

**原因**：铂力特是商业航天板块成员，其定价受板块情绪影响。

- 绝对涨跌 = 大盘因素 + 板块因素 + 个股因素
- 相对强弱 = 个股因素（剔除大盘和板块）

**设计**：
```
relative_strength = anchor_return - sector_avg_return
```

### 2.2 为什么设计"双层股票池"

**原因**：不同股票与铂力特的关联度不同。

| 层级 | 用途 | 参与均值计算 |
|------|------|-------------|
| core_universe | 交易层对标（情绪联动强） | 是 |
| research_core | 研究层对标（产业链相关） | 否 |
| extended_universe | 扩展观察（相关性弱） | 否 |

### 2.3 为什么设计"连续观察层"

**原因**：单日数据噪音大，需要近5日数据判断趋势。

- 连续跑赢板块 > 单日跑赢
- 连续缩量 > 单日缩量
- 资金连续流入 > 单日流入

## 3. 指标设计逻辑

### 3.1 指标分层架构

```
L1: 基础数据（Tushare 原始字段）
    ↓
L2: 衍生指标（计算得出）
    ↓
L3: 状态标签（规则化映射）
    ↓
L4: 综合诊断（多维度综合）
```

### 3.2 核心指标设计

#### 3.2.1 价格强度标签

**输入**：涨跌幅、相对强弱、板块排名
**规则**：
- 强：涨幅>2% 或 (相对强弱>1% 且 排名前半)
- 弱：跌幅>2% 或 (相对强弱<-1% 且 排名后半)
- 中：其余

**设计理由**：避免主观判断，规则固化。

#### 3.2.2 成交额强度标签

**输入**：成交额/5日均值、是否创20日新高、板块排名
**规则**：
- 强：创20日新高 或 成交额>1.5倍均值 或 排名前半
- 弱：成交额<0.7倍均值 且 未创新高 且 排名后半
- 中：其余

**设计理由**：成交额反映市场关注度。

#### 3.2.3 综合信号标签

**输入**：价格强度 × 成交额强度矩阵

| 价格\成交额 | 强 | 中 | 弱 |
|------------|-----|-----|-----|
| 强 | 强 | 中性偏强 | 中性偏强 |
| 中 | 中性偏强 | 中性 | 中性偏弱 |
| 弱 | 中性偏弱 | 中性偏弱 | 弱 |

**设计理由**：价格与量能相互验证。

#### 3.2.4 近5日结构

**输入**：最近5日的相对强弱序列、成交额序列、资金流向
**输出**：
- 跑赢板块 X/5 日
- 平均相对强弱
- 量能状态（连续放量/连续缩量/平稳）
- 资金状态（偏强/有改善/偏弱）

**设计理由**：判断趋势连续性，避免单日噪音。

### 3.3 诊断层设计

**输入**：当日指标 + 近5日指标
**输出**：
- 诊断标签（中性偏强，下行压力仍存）
- 诊断原因（跑输板块、近5日结构未坏、主力未确认回流）
- 明日观察清单（主力是否转正、研究层对标是否继续走弱）
- 信号拆解（异常/积极/风险）

**设计理由**：结构化输出，便于快速决策。
